Documentation

Artemis is a frosted-glass Roblox UI library. One ModuleScript, zero dependencies, fully themeable with any accent color.

✦

All components live inside a Section, inside a Tab, inside a Window. This three-level hierarchy keeps your UI organized and the code readable.

🆕

v1.4 adds: MultiDropdown component, SetAccent (with rainbow mode), SetTheme / GetThemes, and mobile FAB for touch devices. v1.3 added built-in search overlay, ColorPicker, KeySystem, FadeToggle, and AltIconLibrary.

Installation

Reference ArtemisUI using loadstring from the official URL.

local ArtemisUI = loadstring(game:HttpGet("https://artemis.valencea.xyz/loader.lua"))()

Quick Start

local ArtemisUI = loadstring(game:HttpGet("https://artemis.valencea.xyz/loader.lua"))()

local win = ArtemisUI:Window({
    Title            = "My Panel",
    Subtitle         = "v1.0",
    Accent           = Color3.fromRGB(108, 99, 255),
    LoadingAnimation = 2,
})

local sec = win:Tab("#home Main"):Section("#zap Combat")

sec:Toggle("Enabled", false, function(v)
    print("toggled:", v)
end)

Window

ArtemisUI:Window(cfg: table) → WinObj
Creates and animates in a new window. Only one window should exist at a time. The returned WinObj also exposes a .Flags table containing all current flag values, identical to what :LoadedFlags() returns.
FieldTypeDefaultDescription
Titlestring"ArtemisUI"Window title shown in the topbar.
Subtitlestring""Small subtitle shown after a · separator in the topbar.
AccentColor3RGB(108,99,255)Accent color applied everywhere: glows, borders, pills, indicators.
IconstringnilLucide icon name displayed next to the title in the topbar.
ConfigFilestringnilFilename (no extension) for JSON config saving. Enables Flags and auto-save. File is written to ArtemisUI/<name>.json.
LoadingAnimationnumber11 = fullscreen splash overlay. 2 = window-sized morph card. Any other value skips the animation entirely.
FadeTogglebooleanfalseWhen true, the window hides/shows with a fade overlay instead of shrinking to zero size.
AltIconLibrarystringnilURL to a custom icon library. Use the @ prefix on labels to pull icons from this library.
KeySystemtablenilEnables a key validation modal before the window appears. See KeySystem.
ⓘ

You can set getgenv().Scale = 1.1 before creating the window to override the auto-scale that Artemis computes from the viewport size.

KeySystem

Pass a KeySystem table in the window config to show a blocking modal before the main window opens. The user must pass validation to proceed.

FieldTypeDefaultDescription
Titlestring"Key Required"Title text of the modal card.
Subtitlestring"Enter your key to continue."Instruction text below the title.
Placeholderstring"Paste key here..."Placeholder text for the key input box.
GetKeyUrlstringnilIf provided, a "Get Key →" button opens this URL in the browser via GuiService.
OnValidatefunction(keyInput)nilReceives a KeyInput object. Call :Get() on it to read the entered text. Return true to accept, anything else to reject (shakes the card and shows an error).
local win = ArtemisUI:Window({
    Title = "Premium Panel",
    KeySystem = {
        Title      = "License Key",
        Subtitle   = "Enter your key to continue",
        GetKeyUrl  = "https://yourstore.com/buy",
        OnValidate = function(input)
            return input:Get() == "secret123"
        end
    }
})

Loading Animation

Type 1 — Fullscreen Splash

A separate fullscreen overlay covers the entire screen. The title sweeps in from below, a progress bar fills, then everything fades and the main window opens. The overlay uses a separate ScreenGui at DisplayOrder 1000 so it sits above everything.

Type 2 — Window Morph

A card the exact size of the window bounces in from zero using a Back easing. The loading sequence plays inside it, then it seamlessly transitions into the actual window with no positional jump.

No Animation

Pass any value other than 1 or 2 (e.g. 0) to skip the loader entirely and go straight to the KeySystem check (or the window itself).

Tab

win:Tab(name: string) → TabObj
Adds a button to the tab bar and a scrollable content pane. The first tab created is auto-activated. Tab buttons support icon prefixes.

Prefix the name with #iconname for a Lucide icon or @iconname for an icon from the AltIconLibrary. The icon and the display text are parsed automatically — the full string including the prefix is used as the internal tab key.

Section

tab:Section(name: string) → SecObj
Groups components inside a labeled card with an accent top-line and automatic dividers between rows. Supports the same # and @ icon prefixes as tabs. The card border brightens on hover.

Button

sec:Button(text: string, callback: function)
Clickable row with a chevron pill on the right. Plays a ripple on press. If the callback throws, the row briefly flashes red and shows "Error" before restoring. Auto-registered in the search overlay — clicking the result fires the callback directly.

Toggle

sec:Toggle(text: string, default: boolean, callback: function, opts?: table) → ToggleObj
Animated pill toggle switch. The knob uses a Back easing for a satisfying snap. Clicking the result in the search overlay flips the toggle directly.
Return methodDescription
:Set(bool)Explicitly set the state. Animates and fires the callback unless value is unchanged.
:Get()Returns current boolean state.
:Toggle()Flips the current state.

Options: { Flag = "string" } — auto-saves and restores the boolean when ConfigFile is set.

Slider

sec:Slider(text: string, min: number, max: number, default: number, callback: function, opts?: table) → SliderObj
Draggable slider with a live value pill. The knob grows on press and snaps back on release. The fill track uses a gradient overlay for a subtle depth effect.
Return methodDescription
:Get()Returns the current numeric value.
:Set(number)Sets the value, updates visuals, and fires the callback.

Options: { Flag = "string", Increment = number } — Increment snaps values to multiples (e.g. 5 gives steps of 5). Defaults to 1.

ColorPicker

sec:ColorPicker(text: string, default: Color3, callback: function, opts?: table) → ColorPickerObj
Clicking the swatch opens a floating HSV panel with a saturation/value square, a hue slider, a hex input, and a live preview chip. The panel positions itself intelligently to stay within the window bounds and flips above the row if there isn't room below. Clicking outside closes it.
Return methodDescription
:Get()Returns the current Color3.
:Set(Color3)Sets the color, updates all picker visuals, and fires the callback.

Options: { Flag = "string" } — saves and restores the color as a {R, G, B} array (0–1 range) in the config file.

sec:ColorPicker("ESP Color", Color3.fromRGB(255,100,100), function(c)
    print("Selected:", c)
end, { Flag = "espColor" })

Dropdown

sec:Dropdown(text: string, options: {string}, callback: function, opts?: table) → DropObj
Animated expanding dropdown. Includes a built-in search box that filters options in real-time. Selecting an option closes the dropdown, updates the header label, and fires the callback. Only one value can be selected at a time — see MultiDropdown for multi-select.
Return methodDescription
:Get()Returns the currently selected string.
:Set(string)Sets the selection and updates visuals. Does not fire the callback.
:Update({string})Replaces the options list entirely. Resets selection to the first new option.

Options: { Flag = "string" } — auto-saves the selected string.

MultiDropdown v1.4

sec:MultiDropdown(text: string, options: {string}, defaults: {string}, callback: function, opts?: table) → MultiDropObj
Same layout and animation as Dropdown but allows multiple selections simultaneously. Each option row has a checkmark box that toggles independently. The header label summarizes the current selection: shows the single name if one is selected, N selected for a partial set, All when everything is checked, and None when empty. The callback fires on every individual toggle with the full current selection as an ordered array.
Return methodDescription
:Get()Returns an ordered {string} array of all currently selected options, in the original options order.
:Set({string})Replaces the entire selection. Rebuilds visuals and fires the callback.
:SetOptions({string})Replaces the options list. Clears selection and resets the label to None.
:Clear()Deselects everything, updates the label, and fires the callback with an empty table.

Options: { Flag = "string" } — saves and restores the selection as a JSON array of strings.

local layers = sec:MultiDropdown("Visible Layers", {
    "Players", "NPCs", "Props", "Effects"
}, { "Players", "Props" }, function(selected)
    for _, v in ipairs(selected) do print(v) end
end, { Flag = "visibleLayers" })

print(layers:Get())   -- { "Players", "Props" }
layers:Clear()       -- deselects all, fires callback with {}

Input

sec:Input(label: string, placeholder: string, callback: function) → InputObj
Text input box with a small uppercase label above it. The callback receives (text: string, pressedEnter: boolean) when focus is lost. The box border highlights with the accent color on focus.
Return methodDescription
:Get()Returns the current text content.
:Set(string)Sets the text content directly.

Keybind

sec:Keybind(text: string, default: Enum.KeyCode | string, callback: function) → KeybindObj
Click the pill to enter binding mode, then press any key or mouse button (MB1, MB2, MB3) to assign it. The bound input fires the callback globally anywhere in-game. Pass a KeyCode enum or a string like "F" or "MB2" as the default.
Return methodDescription
:Get()Returns the current keyData table: { mouse, keyCode?, inputType?, name }.
:Set(value)Sets the bind from a KeyCode or string. Falls back to the default if unresolvable.

Paragraph

sec:Paragraph(header: string, body: string) → ParagraphObj
Static text block. Pass an empty string for header to omit it. The section card resizes automatically when the body changes. Useful for live status displays.
Return methodDescription
:Update(header, body)Replaces both header and body at once. Creates a header label if one didn't exist and a non-empty string is passed.
:SetHeader(text)Updates only the header. No-op if no header was created at init.
:SetBody(text)Updates only the body text. The card height adjusts automatically.

Separator

sec:Separator()
Thin horizontal rule inside the section card. Useful for grouping related rows without creating a new section.

Notify

win:Notify(title: string, body: string, duration: number, side: string)
Delegates to an external notification module (ValenceUI). side is "left" or "right". Multiple notifications stack vertically and shift when one is dismissed.

SetToggleKey

win:SetToggleKey(key: Enum.KeyCode)
Assigns a keyboard shortcut to show/hide the window. Full UI state is preserved between toggles. Hides via the same animation path as the close button.
win:SetToggleKey(Enum.KeyCode.RightShift)

SetAccent v1.4

win:SetAccent(color: Color3 | "rainbow")
Changes the accent color across the entire UI at runtime. Pass a Color3 to set a fixed color (also saves it to the config file if ConfigFile is set). Pass the string "rainbow" to start a Heartbeat-driven HSV cycle. Calling SetAccent with a Color3 while rainbow is active stops the rainbow loop first.
win:SetAccent(Color3.fromRGB(255, 80, 120))
win:SetAccent("rainbow")

SetTheme v1.4

win:SetTheme(name: string)
Switches the window's background and border color palette to a built-in theme. Updates all background frames, strokes, and scrollbar colors immediately. Also saves the theme name to the config file under the __theme key so it restores on next load.

Built-in themes: "Valence", "Midnight", "Obsidian", "Dusk", "Carbon", "Ember".

win:SetTheme("Midnight")
win:SetTheme("Ember")

GetThemes v1.4

win:GetThemes() → {string}
Returns a sorted array of all built-in theme names. Useful for populating a Dropdown that lets users switch themes at runtime.
sec:Dropdown("Theme", win:GetThemes(), function(v)
    win:SetTheme(v)
end)

Search

A ⌕ button sits in the topbar to the left of the minimize button. Clicking it opens a fullscreen overlay on top of the content area with an auto-focused search input. Every component registers itself automatically when created. Clicking a result activates it directly — toggles flip, buttons fire their callbacks, dropdowns scroll to the relevant tab and section.

Press Escape to close the overlay. The search state does not persist between open/close cycles.

Save

win:Save()
Manually writes all current flag values to disk as JSON. Called automatically whenever a flagged component changes. Requires ConfigFile to be set and writefile to be available in the executor environment.

LoadConfig

win:LoadConfig(fileName: string)
Loads a different config file (no extension) into the flags table and pushes all values through their registered flagHandlers, applying them to the live UI. Useful for profile/preset switching.

LoadedFlags

win:LoadedFlags() → table
Returns the internal flags table. You can also reference it directly via win.Flags. The table is shared by reference — changes made to it externally will be reflected in the next :Save() call.

Destroy

win:Destroy()
Scales the window down and destroys the ScreenGui after the animation completes. Disconnects the rainbow Heartbeat loop if active and stops the topbar pulse animation.

Icons

Artemis uses the full Lucide icon set (48px spritesheet) by default. Prefix any label string with #iconname to show that icon. Use @iconname to load from the AltIconLibrary instead.

Icon prefixes work on Tab names, Section names, Button text, Toggle text, Slider text, Dropdown text, MultiDropdown text, ColorPicker text, and Keybind text.

✦

Icons load asynchronously. The library fetches over HTTP in a separate thread on startup. Icons render within a few frames of the UI appearing — no pop, no layout shift.

Config Saving

Set ConfigFile = "myfile" on the window and pass { Flag = "flagName" } to any Toggle, Slider, Dropdown, MultiDropdown, or ColorPicker. Values auto-save to ArtemisUI/myfile.json on every change and restore on the next load. The theme (__theme) and accent color (__accent) are also stored in the same file.

If the file contains malformed JSON it is silently reset to {} so it never blocks startup.

FadeToggle

When FadeToggle = true in the window config, hiding and showing the window uses a full-tree transparency fade instead of a size animation. Every TextLabel, ImageLabel, UIStroke, and Frame in the window hierarchy is individually tweened. The window position and size are fully preserved between toggles.

Alt Icon Library

Pass AltIconLibrary = "https://your-cdn.com/icons.lua" to the window config. The URL must return a Lua table in the same spritesheet format as the default Lucide set ({ ["48px"] = { iconname = { id, {w,h}, {ox,oy} } } }). Use the @iconname prefix anywhere to pull from this library.

Mobile FAB v1.4

On touch-enabled devices, closing the window reveals a draggable Floating Action Button (FAB) in the left-center of the screen. Tapping it reopens the window. The FAB remembers its last dragged position between open/close cycles within the same session.

A separate MobileBtn is also available at the left edge as an alternate re-open target. Both buttons support drag-to-reposition — a tap that doesn't move more than 5–6px is treated as a tap, not a drag.

⚠

The FAB only appears when UserInputService.TouchEnabled is true. On desktop it stays hidden regardless of how the window is closed.

Full Example Script

Every component, theme switching, accent control, MultiDropdown, search, paragraph updates, KeySystem, and config saving in one script.

local ArtemisUI = loadstring(game:HttpGet("https://artemis.valencea.xyz/loader.lua"))()

local win = ArtemisUI:Window({
    Title            = "ArtemisUI",
    Subtitle         = "v1.4",
    Accent           = Color3.fromRGB(108, 99, 255),
    Icon             = "sword",
    ConfigFile       = "artemis_demo",
    LoadingAnimation = 2,
    FadeToggle       = true,
    KeySystem        = {
        Title      = "Access Required",
        Subtitle   = "Enter the key from our Discord",
        GetKeyUrl  = "https://discord.gg/example",
        OnValidate = function(input) return input:Get() == "demo123" end
    }
})

win:SetToggleKey(Enum.KeyCode.RightShift)
win:Notify("ArtemisUI", "Loaded. Press ⌕ to search.", 4, "right")

-- Main tab
local main = win:Tab("#home Main")
local combat = main:Section("#sword Combat")

local aura = combat:Toggle("Aura Active", false, function(v)
    print("Aura:", v)
end, { Flag = "auraActive" })

combat:Keybind("Toggle Aura", Enum.KeyCode.F, function()
    aura:Toggle()
    win:Notify("Aura", aura:Get() and "Enabled" or "Disabled", 2, "right")
end)

-- Multi-select gear selection
local selectedGears = {}
combat:MultiDropdown("Active Gears", {
    "Landmine", "AutoTurret", "Cloak", "Shuriken"
}, {}, function(v)
    selectedGears = v
end, { Flag = "activeGears" })

local movement = main:Section("#zap Movement")
local speed = movement:Slider("Walk Speed", 16, 300, 16, function(v)
    local c = game.Players.LocalPlayer.Character
    if c and c:FindFirstChild("Humanoid") then c.Humanoid.WalkSpeed = v end
end, { Flag = "walkSpeed", Increment = 5 })

-- Status paragraph with live updates
local statusSec = main:Section("#info Status")
local infoP = statusSec:Paragraph("Session", "Idle")
local kills = 0
statusSec:Button("Simulate Kill", function()
    kills += 1
    infoP:SetBody("Running — " .. kills .. " kills")
end)

-- Settings tab
local settings = win:Tab("#settings Settings")

local appearance = settings:Section("#palette Appearance")
appearance:Dropdown("Theme", win:GetThemes(), function(v)
    win:SetTheme(v)
end)
appearance:ColorPicker("Accent", Color3.fromRGB(108,99,255), function(c)
    win:SetAccent(c)
end, { Flag = "customAccent" })
appearance:Toggle("Rainbow Mode", false, function(v)
    win:SetAccent(v and "rainbow" or Color3.fromRGB(108,99,255))
end)

local perf = settings:Section("#gauge Performance")
local fps = perf:Slider("FPS Cap", 30, 240, 60, function(v)
    setfpscap(v)
end, { Flag = "fpsCap", Increment = 10 })

perf:Button("Reset All", function()
    speed:Set(16); fps:Set(60); aura:Set(false)
    win:Save()
    win:Notify("Reset", "All settings restored.", 3, "right")
end)
